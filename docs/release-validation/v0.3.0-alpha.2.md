## Live Lima validation

The macOS ARM64 archive was extracted without installing it over the local development build. Its
packaged Linux ARM64 `lima-guest/guard-helper` was copied to a unique temporary path under
`/usr/local/bin` in the existing mountless `sandbox-guard` Lima instance, then the packaged macOS
`guard` binary was run with:

```text
guard test --backend macos-lima --require-cgroup --helper <temporary-guest-helper>
```

The exact packaged pair passed:

```text
backend: MacosLima
filesystem boundary: ok
environment boundary: ok
PID namespace: ok
host loopback isolation: ok
controlled egress denial and direct-bypass isolation: ok
seccomp namespace and process-memory denial: ok
seccomp thread compatibility: ok
trusted supervisor memory: protected
trusted preflight sequencing: ok
rlimits: ok
cgroup v2: enforced
```

The temporary guest helper copies were removed after the test. Lima emitted stale SSH
multiplex-socket fallback warnings, but every validation assertion passed and the command exited
with status 0.
