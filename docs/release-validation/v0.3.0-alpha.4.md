# v0.3.0-alpha.4 release validation

**Status:** published as an alpha prerelease on 2026-07-16 (UTC).

## Security purpose

This release fixes the bwrap launcher-environment disclosure documented after alpha.2 and alpha.3.
Bubblewrap's `--clearenv` cleared the executed child environment but not bwrap's own inherited
environment; bwrap remains pid 1 inside the sandbox PID namespace, where a confined tool could read
`/proc/1/environ`. Alpha.4 places a fixed absolute `/usr/bin/env -i` boundary immediately before
bwrap on every Linux-host and Lima-guest route, leaving the launcher environment empty. The hostile
backend probe now reads `/proc/1/environ` and fails unless it is empty, while independently checking
that the explicit child `HOME`, `PATH`, and `LANG` environment survives.

The published alpha.2 and alpha.3 GitHub release descriptions were updated with the known-issue and
upgrade guidance before alpha.4 was published. The versioned alpha.2/alpha.3 validation records also
explain why their historical child-environment probe did not detect the pid-1 disclosure.

## Live Lima validation

The exact published macOS ARM64 `guard` binary
(`033ff5fa41ca47353419412000ba7ff94a9e1ee7b5e4cc299f3c7de0d3aa4e4f`) and published Linux ARM64
`guard-helper` (`47597fb37418982a5a67d35e451da136da1c866cf80bc89022688dba28962bf9`) were extracted without
replacing installed binaries. The helper hash matches both the Linux ARM64 archive and the macOS
archive's `lima-guest/guard-helper` entry.

The helper was copied to a unique temporary path in the existing mountless `sandbox-guard` Lima
instance, hash-verified in the guest, and used with:

```text
guard test --backend macos-lima --require-cgroup --helper <temporary-guest-helper>
```

All probes passed with exit status 0: filesystem boundary, child environment boundary, **empty
bwrap launcher `/proc/1/environ`**, PID namespace, host loopback isolation, controlled-egress denial
and direct-bypass isolation, seccomp namespace/process-memory denial and thread compatibility,
trusted-supervisor memory, preflight sequencing, rlimits, and required cgroup v2 enforcement. The
temporary helper and upload staging file were removed afterward, and their removal was confirmed.
